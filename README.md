# Audio Scriber

Audio Scriber is an AI-powered meeting transcription and summarization application. It provides real-time audio recording, live transcription using Deepgram, and intelligent session summaries generated by Google Gemini.

## üöÄ Features

- **Real-time Transcription**: Live speech-to-text conversion using Deepgram's Nova-2 model.
- **AI Summaries**: Intelligent meeting summaries with action items and key decisions using Google Gemini 2.0 Flash.
- **Session Management**: Record, save, and review past sessions.
- **Live Audio Visualization**: Real-time waveform visualization during recording.
- **Authentication**: Secure user authentication via Better Auth (Google OAuth).

## ‚öôÔ∏è Installation & Setup

### 1. Clone the Repository
```bash
git clone <repository-url>
cd audio-scriber
```

### 2. Environment Variables
Create a `.env` file in the root directory (or configure your Docker environment) with the following variables:

```env
# Database
DATABASE_URL="postgresql://user:password@postgres:5432/audioscriber?schema=public"

# Backend
PORT=8000
BETTER_AUTH_SECRET="your_random_secret_string"
BETTER_AUTH_URL="http://localhost:8000"

# AI Services
DEEPGRAM_API_KEY="your_deepgram_api_key"
GEMINI_API_KEY="your_google_gemini_api_key"

# Google OAuth (for Better Auth)
GOOGLE_CLIENT_ID="your_google_client_id"
GOOGLE_CLIENT_SECRET="your_google_client_secret"

# Frontend
NEXT_PUBLIC_API_URL="http://localhost:8000"
```

### 3. Run with Docker (Recommended)
The easiest way to run the application is using Docker Compose.

```bash
docker-compose up --build
```

This will start:
- **PostgreSQL Database** on port `5432`
- **Backend API** on port `8000`
- **Frontend App** on port `3000`

Access the application at `http://localhost:3000`.

### 4. Manual Setup (Dev Mode)

**Backend:**
```bash
cd backend
npm install
npx prisma generate
npm run dev
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

## üîå API Endpoints

### Sessions
- `GET /api/sessions` - List all sessions
- `GET /api/sessions/:id` - Get session details
- `POST /api/sessions` - Create a new session
- `POST /api/sessions/:id/summary` - Generate AI summary

### Real-time
- **Socket.io**: Connects to `/` namespace for live audio streaming and transcription events.
